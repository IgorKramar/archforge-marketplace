# /krait_arch:c4

Generate a C4 diagram in Mermaid at one of the standard levels.

## Inputs

- Arguments: `$ARGUMENTS`. Expect two parts:
  1. Level: `context` (L1), `container` (L2), or `component` (L3).
  2. Subject: free-text description of what to diagram.
- Project context: `./ARCHITECTURE.md` (existing topology), repository structure (look at top-level dirs and key config files for inferring containers).

If the level argument is missing or invalid, ask the user to pick one. Don't render multiple levels in one shot — that defeats the purpose.

## Steps

1. **Use the `c4-diagrams` skill** as the source of truth for notation and rules.

2. **Inspect the repo** briefly to ground the diagram in reality:
   - For Container level: look at `package.json` / `pyproject.toml` / docker-compose / k8s manifests to find services and infrastructure.
   - For Component level: look at the directory structure of the targeted service.
   - For Context level: look at README, ARCHITECTURE.md, package metadata.

3. **Generate Mermaid** following all the C4 rules from the skill:
   - Every arrow labeled (protocol + purpose).
   - Every box names its technology.
   - One level per diagram.
   - Legend if needed.

4. **Save to** `docs/architecture/diagrams/<level>.md`. If a file at that path exists, ask before overwriting and offer to save with a date suffix instead.

5. **Update `ARCHITECTURE.md`** if the high-level structure section refers to this diagram.

## Output to chat

- The Mermaid code block (so the user can render in chat).
- The path it was saved to.
- A note about what's *missing* from the diagram — usually you've inferred from the repo, and the user will know things you don't (e.g., external systems, future containers).

## Discipline

- **Don't invent components**. If the repo doesn't show it and the user didn't tell you, leave it out.
- **Don't render more than 15 elements** at L1/L2. If the system is bigger, render a subset and say which subset.
- **No bidirectional arrows**. Pick the initiator side.
- **No emojis in the diagram**. Mermaid handles them inconsistently and they don't add information.
